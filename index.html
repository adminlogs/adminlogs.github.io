  • Active Directory: GPO

    use the tool gpedit.msc to edit the local GPO.

    • Computer Configuration: to apply the GPO the computer has to be rebooted
    • User Configuration: to apply the GPO the user have to logout/login

    The policies definitions are stored by default in %SystemRoot%\PolicyDefinitions folder, but Windows Server is looking up¬† for policies definitions in %SystemRoot%\SYSVOL\sysvol\domain\Policies first then in %SystemRoot%\PolicyDefinitions. The new policies definitions are put in %SystemRoot%\SYSVOL\sysvol\domain\Policies.

    GPO precedence from lowest to highest: Local Policy ‚ÜíSite Policy ‚ÜíDomain Policy ‚ÜíOU Policy.

    • Local Group Policy settings are applied first.
    • GPOs linked at the site level are applied next followed by the GPOs linked at the domain level and OU level. Since GPOs linked to the OU are processed last, they have the highest precedence.
    • In the case of nested OUs, GPOs linked to the parent OUs are applied first followed by the GPOs linked to the child OU.
    • If multiple GPOs are linked to a container, then the GPO with the lowest link order will have the highest precedence.
    • To view the list of GPOs applied to a container, double-click the container and select the Group Policy Inheritance¬†tab in the right pane. A list of GPOs with link order, location, and status will be displayed.

    Enforcing a GPO:

    GPOs can be enforced so that the GPOs linked to a higher level container like domain or a parent OU takes precedence over the GPOs linked to a lower level container. To enforce a GPO, select the GPO linked to a container. Right-click the GPO and select Enforced.

    GPOs that are enforced will be applied to a lower-level container even when the ‚ÄòBlock Inheritance‚Äô option is enabled for that container.

  • Active Directory: Groups

    Changing Active Directory Group type implies the change of the group SID (Security ID).

    Group scope:

    Changing a group scope from Domain Local to Global or vice versa is not possible directly unless the scope is changed to Universal.

    • Domain Local: can contain domain objects, child domain objects and trusted domain objects. The group cannot be applied to a resources on trusted and child domain but to local domain.
    • Global: contains domain objects only. The group can be applied to resources on trusted, child and local domain.
    • Universal: contains domain objects and child domain objects. The group can be applied to resources on trusted, child and local domain.
    New-Group -Name Name -Path Path -GroupCategory Category -GroupScope Scope

    The global catalog contains a partial representation of all objects in the entire forest. The global catalog is built and updated automatically by the AD replication system. Only certain attributes of the objects which are likely to be searched in the queries are updated to these servers. The global catalog is stored on domain controllers that have been designated as global catalog servers.
    All DCs will contain at least three partitions, which are as follows:

    • The domain partition: This partition contains information about a domain‚Äôs objects and their attributes. This partition is a writable replica.
    • The configuration partition: This partition contains information about the forest‚Äôs topology such as the domain controllers positioning, site links, and more. This partition is a writable replica.
    • The schema partition: This partition contains definitions of every object class in the forest and the rules that control the creation and manipulation of those objects. Every DC will have this partition as read-only, except for the Schema master DC, which will have a writable replica of the schema partition.

    The global catalog servers also would have a separate partition that contains partial information about objects in other domains A global catalog server may have multiple of these partitions depending on the number of domains in the forest. These partitions are read-only partitions.

    NTDS partitions:

    • Application: This partition stores information about applications in an AD. Suppose AD integrated DNS zones information is stored in this partition.
    • Configuration: Just like schema partition, there is just one master configuration partition per forest and a second one on all DCs in a forest. It contains the forest-wide active directory topology including DCs and sites and services. It is replicated to all DCs in a forest.
    • Schema: There is only one schema partition per forest and it is stored in all DCs of the forest. It contains the definition of objects and rules for their manipulation and creation in an active directory. It is replicated to all DCs of the forest.
    • Domain: contains information about users, groups, computers and OUs. It is replicated to all DCs in a given domain. AD Users and Computers.
